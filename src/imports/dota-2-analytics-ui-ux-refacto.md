Внеси правки в текущий дизайн Android-приложения Dota 2 analytics. Это не новый дизайн с нуля, а доработка уже существующего UI/UX с исправлением логики, навигации, структуры экранов и контента.

Важно: приложение проектируется под сборку в APK через Capacitor, с обязательным использованием capacitor.config.ts. Интерфейс должен оставаться adaptive / responsive и корректно работать на любых разрешениях экрана.

Исправить следующие проблемы и недочеты:

1. Удалить дублирование “Безопасность” и “Настройки”
- Сейчас “Безопасность” и “Настройки” дублируют друг друга как отдельные кнопки.
- Удалить кнопку “Безопасность” как отдельный пункт.
- Все, что относится к безопасности, перенести внутрь раздела “Настройки”.


2. Добавить переключение между несколькими Steam-аккаунтами
- Если пользователь добавляет еще один Steam-аккаунт, должна появляться возможность удобного переключения между ними.
- В разделе профиля / привязки Steam-профиля добавить:
  - список подключенных Steam-аккаунтов
  - активный аккаунт
  - кнопку “Сделать активным”
  - кнопку “Переключить”
  - кнопку “Удалить аккаунт”
  - кнопку “Добавить еще аккаунт”
- Показать UI состояния:
  - один подключенный аккаунт
  - несколько аккаунтов
  - активный аккаунт
  - переключение между аккаунтами
- После переключения должен обновляться весь контент приложения:
  - матчи
  - сборки
  - аналитика
  - аккаунт
  - цели

3. Добавить кнопку “Назад” в настройки
- Сейчас при переходе в раздел “Настройки” не появляется кнопка назад, и пользователь застревает внутри экрана.
- Исправить навигацию так, чтобы в “Настройках” всегда была понятная кнопка “Назад”.
- Использовать такой же паттерн навигации, как и в остальных внутренних экранах приложения.
- В верхней панели каждого внутреннего подраздела должна быть:
  - кнопка назад
  - заголовок текущего раздела
  - при необходимости кнопка сохранения или меню действий

4. Исправить отображение роли support
- Сейчас роль support показана некорректно.
- Разделить support на:
  - Position 4
  - Position 5
- Обновить это во всех местах интерфейса:
  - фильтры матчей
  - аналитика по ролям
  - карточки матчей
  - профиль аккаунта
  - рекомендации по сборке
  - сравнение
- Использовать понятные подписи:
  - Support 4
  - Hard Support 5
  или
  - Pos 4
  - Pos 5
- Сделать соответствующие иконки / теги для этих ролей

5. Исправить логику блока “Сборки”
- В разделе “Сборки” не должно быть случайного, бессмысленного или шаблонного текста.
- Сборки должны основываться на анализе матчей пользователя и помогать ему улучшать игру.
- UI должен показывать, что рекомендации формируются на основе:
  - последних матчей
  - выбранного героя
  - роли
  - позиции
  - патча
  - винрейта
  - частых ошибок игрока
- Для каждой рекомендации по сборке показывать:
  - почему она предлагается
  - какие ошибки обнаружены в прошлых матчах
  - что стоит изменить
  - recommended item progression
  - situational items
  - skill build
  - talents
- Формулировки должны быть осмысленными, полезными и аналитическими, а не общими фразами.
- Добавить блоки:
  - “Что ты делаешь хорошо”
  - “Что стоит улучшить”
  - “Почему эта сборка лучше для твоих матчей”
  - “Частые ошибки в таймингах и предметах”

6. Убрать полностью подписку
- Полностью удалить весь раздел “Подписка”.
- Удалить из интерфейса:
  - тарифы
  - premium / pro
  - paywall
  - платные функции
  - upsell-блоки
  - pay buttons
- Сервис сейчас полностью бесплатный.
- Все экраны и блоки должны выглядеть без монетизации.
- Если где-то подписка встроена в профиль, настройки или личный кабинет, убрать ее полностью и перестроить layout без пустот.

7. Исправить неработающие кнопки
Следующие кнопки сейчас не работают и должны быть полноценно встроены в дизайн и пользовательский flow:
- Экспорт данных
- Привязанные аккаунты
- Список устройств

Для каждой из этих кнопок нужно сделать полноценные рабочие экраны / состояния:

7.1 Экспорт данных
- Экран экспорта
- Форматы:
  - PDF
  - CSV
- Что можно экспортировать:
  - общую статистику
  - матчи
  - аналитику
  - сборки
- Добавить:
  - выбор диапазона дат
  - выбор разделов для экспорта
  - состояние загрузки
  - состояние успешного экспорта
  - состояние ошибки

7.2 Привязанные аккаунты
- Экран со списком привязанных Steam-аккаунтов
- Показать:
  - аватар
  - nickname
  - steam profile URL
  - активный аккаунт
  - дата последней синхронизации
- Действия:
  - добавить аккаунт
  - переключить активный
  - удалить
  - обновить данные
- Добавить empty state для случая, если аккаунтов нет

7.3 Список устройств
- Экран устройств пользователя
- Показать:
  - название устройства
  - платформа
  - дата последнего входа
  - текущее устройство
- Действия:
  - выйти с устройства
  - выйти со всех устройств
- Добавить подтверждение для destructive actions

8. Обновить источник ассетов и иконок
Использовать в проекте в качестве основной базы:
- OpenDota web assets
- OpenDota / dotaconstants

Важно:
- OpenDota / dotaconstants использовать как основу проекта для ID, имен, slug’ов и соответствий для API
- OpenDota web assets использовать для игровых изображений и иконок
- В UI явно учитывать, что это основные источники статических игровых данных и медиа

Организация игровых ассетов должна быть такой:

/assets/dota/heroes/ — портреты героев
/assets/dota/items/ — предметы
/assets/dota/abilities/ — скиллы
/assets/dota/ranks/ — медали и ранги
/assets/dota/roles/ — роли: carry, mid, offlane, pos4, pos5
/assets/dota/lanes/ — mid, safe, offlane, jungle
/assets/dota/ui/ — Radiant, Dire, victory, defeat, patch, region icons
/assets/dota/aspects/ — аспекты героев
/assets/dota/facets/ — facets / геройские модификаторы, если используются отдельно
/assets/dota/neutral-items/ — нейтральные предметы
/assets/dota/status/ — silence, stun, root, mute, break, hex, invisibility, dispel и другие игровые статусы
/assets/dota/objectives/ — tower, barracks, tormentor, roshan, lotus, watchers, wisdom rune, bounty rune, power rune
/assets/dota/modes/ — ranked, turbo, normal, captains mode и другие режимы
/assets/dota/attributes/ — strength, agility, intelligence, universal
/assets/dota/minimap/ — элементы миникарты, пинги, объекты, руны
/assets/dota/badges/ — streak, MVP-like badges, performance icons
/assets/dota/currency/ — gold, net worth, buyback, unreliable gold
/assets/dota/stats/ — damage, healing, xp, last hits, denies, vision, wards, deward, smoke, dust
/assets/dota/patches/ — патчи и их обозначения

- Обязательно добавить в дизайн поддержку аспектов / facets / игровых модификаторов героев, если они используются в анализе.

После внесения всех изменений:
- проверить, чтобы не осталось тупиковых экранов без кнопки возврата
- проверить, чтобы не осталось неработающих кнопок
- проверить, чтобы не осталось пустых разделов после удаления подписки
- проверить, чтобы переключение между Steam-аккаунтами было понятным и быстрым
- проверить, чтобы в “Сборках” контент выглядел как аналитический помощник, а не как случайно сгенерированный текст
- обновить все связанные экраны, компоненты, состояния и navigation flow

Что нужно обновить в макетах:
- экран профиля
- экран настроек

- экран экспорта данных
- экран устройств

Итоговое требование:
Сделай дизайн более логичным, целостным и продуктовым. Исправь навигационные ошибки, убери лишние сущности, реализуй рабочие пользовательские сценарии, обнови структуру аккаунтов и синхронизации, а рекомендации по сборкам сделай полезными и основанными на анализе матчей пользователя.
